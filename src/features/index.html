Features

Built-in watchdog

Ubic includes ubic.watchdog service out of the box.

Global service tree

All services are accessible with a single command. For example, ubic status will show the statuses of every service.

Custom status checks

Running process is not necessarily a working process. Write custom code for additional status checks.

Polymorphic service classes

Encapsulate your service-specific code in class and reuse it everywhere. Anything that looks like service can be started with ubic (example: init scripts). And then you can benefit from ubic's common infrastucture: pretty command-line frontend, watchdog, http status check server, etc.

Code reuse

Don't repeat yourself! Encapsulate your service-specific code and project policies in a base class and reuse it.

Populate service tree dynamically

If you need to run many services, you don't have to write separate config file for each one. Instead, you can describe all of them at once with multiservice.
